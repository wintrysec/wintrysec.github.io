免杀研究

查杀方式

传统的防病毒产品使用签名来识别已知的恶意软件。

如果我们将含有已知特征的shellcode嵌入可执行文件中，则防病毒产品会识别该shellcode并将该可执行文件标记为恶意文件。通过混淆shellcode，可以规避使用简单字符串搜索来识别恶意代码的防病毒产品。

但更多的防病毒产品模拟虚拟沙箱中可执行文件的执行。

在执行的每个模拟步骤中，防病毒产品都会在模拟进程空间中检查已知的恶意程序。如果发现已知特征，则防病毒产品会将可执行文件或DLL标记为恶意。这项技术击败了许多编码器和打包程序，它们试图从基于签名的防病毒产品中隐藏已知的不良信息。

基于签名的检测：静态检查文件校验和(MD5, SHA1等)、二进制文件中是否存在特定字符串或字节序列

启发式检测：对软件的行为进行静态分析，识别潜在的恶意特征(比如使用了与恶意软件有关的特定函数)

虚拟沙箱检测：对软件进行动态分析，在一个受控环境(沙箱)中执行该软件，并对其行为进行监控

AES加密免杀shellcode

1、对抗特征查杀（静态）

服务端用AES动态加密shellcode，客户端用加载器请求并执行shellcode

2、对抗行为分析（静态）

比如加载器不能使用CreateThread等创建线程的API函数，寻找替代方法

3、对抗虚拟沙箱（动态）

避免在沙箱、逆向分析时有明显的恶意行为，使用条件语句判断环境发现并绕过沙箱

触发条件：

1）判断物理内存大小（一般正常虚拟服务器最小2GB）

2）判断硬盘大小（>=50GB）

3）判断是否存在指定进程

可以找Chome等进程，也可自己上传一个无害的exe执行进程(aaa123.exe)

Artifact kit开发套件免杀

Cobalt Strike 使用 Artifact Kit 生成可执行文件和DLL。

Artifact Kit是一个源代码框架，用于构建可逃避某些防病毒产品的可执行文件和DLL。

原理

防病毒沙箱具有局限性，它不是完整的虚拟机，防病毒沙箱不会模仿某些系统行为。 Artifact Kit是可执行文件和DLL模板的集合，这些模板依赖于防病毒产品无法模仿的某些行为来恢复位于二进制文件内部的Shellcode。

Artifact Kit中的src-common/bypass-pipe.c生成可执行文件和DLL，这些可执行文件和DLL通过命名管道为其自身提供shellcode。如果防病毒沙箱不模拟命名管道，则它将找不到已知的Shellcode特征。

局限性

当然，防病毒产品有可能击败Artifact Kit的特定实现。如果防病毒供应商为你使用的Artifact Kit技术编写了签名，则将捕获其创建的可执行文件和DLL。你应该使用其中的一种技术来构建自己的Artifact Kit实现。

即使这样还不够，大部分防病毒产品会镜像系统流量上传样本到反病毒厂商的服务器，他们会分析未知的可执行文件或DLL。

在这种情况下，”混淆”将毫无用处。尝试找到一个已知的好的（白名单）程序（例如powershell），利用此程序会将payload注入内存。

如何使用Artifact kit

1、修改免杀技术

修改src-common/文件夹中使用的免杀技术

2、设置构建选项

修改build.sh文件中的构建选项（源文件和可执行文件的位置）

运行build.sh 开始生成可执行文件和cna脚本

Artifact Kit构建脚本会为每种Artifact Kit技术创建一个包含模板控件的文件夹

3、引入生成的artifact.cna脚本，直接创建新的artifact.exe即可。
